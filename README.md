# CS686 Assignment 11 — Terraform + Ansible Configuration Management

## Overview

This project extends the previous AWS Terraform infrastructure to provision **6 mixed-OS EC2 instances** (3 Ubuntu + 3 Amazon Linux) and introduces an **Ansible Controller** instance that runs a playbook to update packages, verify Docker, and report disk usage across all 6 nodes.

| Tool | Purpose |
|------|---------|
| **Packer** | Builds a custom Amazon Linux 2023 AMI with Docker pre-installed |
| **Terraform** | Provisions VPC, bastion, Ansible controller, and 6 mixed-OS private EC2 instances using reusable modules |
| **Ansible** | Runs a configuration management playbook on all 6 instances from the controller |

## Part A — Build the Custom AMI with Packer

The Packer template produces an Amazon Linux 2023 AMI with Docker pre-installed. This AMI is used for the Amazon Linux instances, the bastion, and the Ansible controller.

```bash
cd packer

packer init amazon-linux.pkr.hcl
packer validate amazon-linux.pkr.hcl
packer build amazon-linux.pkr.hcl
```

After a successful build, note the **AMI ID** from the output — you will need it for `terraform.tfvars`.

> Ubuntu instances use the latest official Ubuntu 22.04 LTS AMI, resolved automatically by Terraform via a data source (no Packer build needed for Ubuntu).

## Part B — Provision Infrastructure with Terraform

### Resources provisioned

| Resource | Count | Details |
|----------|-------|---------|
| VPC | 1 | `10.0.0.0/16` |
| Public subnets | 2 | `10.0.1.0/24`, `10.0.2.0/24` |
| Private subnets | 2 | `10.0.10.0/24`, `10.0.11.0/24` |
| Internet Gateway | 1 | |
| NAT Gateway | 1 | Enables outbound internet for private instances |
| Bastion host | 1 | Public subnet — SSH restricted to your IP |
| Ansible Controller | 1 | Public subnet — Ansible pre-installed, SSH restricted to your IP |
| Ubuntu EC2 instances | 3 | Private subnets — tagged `OS: ubuntu` |
| Amazon Linux EC2 instances | 3 | Private subnets — tagged `OS: amazon` |

### Steps

#### 1. Edit `terraform/terraform.tfvars`

```bash
cd terraform
```

Open [terraform/terraform.tfvars](terraform/terraform.tfvars) and set:

```hcl
key_name      = "your-key-pair-name"     # EC2 Key Pair already in AWS
my_ip_cidr    = "YOUR.IP.HERE/32"        # curl https://checkip.amazonaws.com/
custom_ami_id = "ami-XXXXXXXXXXXXX"      # AMI ID from Packer output
```

#### 2. Initialize Terraform

```bash
terraform init
```

#### 3. Preview the plan

```bash
terraform plan
```

#### 4. Apply

```bash
terraform apply
```

Type `yes` when prompted. After ~5 minutes, Terraform prints the outputs and **automatically generates** `ansible/inventory/hosts.ini` with the correct private IPs.

Example output:
```
ansible_controller_public_ip = "3.X.X.X"
bastion_public_ip             = "3.Y.Y.Y"
ubuntu_instance_ips           = ["10.0.10.X", "10.0.11.X", "10.0.10.Y"]
amazon_instance_ips           = ["10.0.10.Z", "10.0.11.Z", "10.0.10.W"]
ssh_to_ansible_controller     = "ssh -A -i <your-key.pem> ec2-user@3.X.X.X"
```

## Part C — Run the Ansible Playbook

### What the playbook does

| Task | Ubuntu | Amazon Linux |
|------|--------|--------------|
| Update & upgrade packages | `apt upgrade -y` | `dnf update -y` |
| Install / update Docker | Docker CE via official apt repo | `dnf install docker` |
| Verify Docker version | `docker version` | `docker version` |
| Report disk usage | `df -h` | `df -h` |

### Steps

#### 1. Add your key to the SSH agent (on your local machine)

```bash
ssh-add /path/to/your-key.pem
```

#### 2. SSH into the Ansible controller with agent forwarding

```bash
ssh -A -i /path/to/your-key.pem ec2-user@<ANSIBLE_CONTROLLER_PUBLIC_IP>
```

Use the `ansible_controller_public_ip` from Terraform output.

#### 3. Copy the inventory and playbook to the controller

From your local machine:

```bash
CONTROLLER_IP=$(cd terraform && terraform output -raw ansible_controller_public_ip)

# Copy inventory (generated by Terraform)
scp -i /path/to/your-key.pem \
    ansible/inventory/hosts.ini \
    ec2-user@$CONTROLLER_IP:~/ansible/inventory/hosts.ini

# Copy playbook and config
scp -i /path/to/your-key.pem \
    ansible/playbook.yml \
    ansible/ansible.cfg \
    ec2-user@$CONTROLLER_IP:~/ansible/
```

#### 4. Run the playbook from the controller

```bash
# On the Ansible controller:
cd ~/ansible
ansible-playbook -i inventory/hosts.ini playbook.yml
```

#### Alternative — Run the playbook directly from your local machine

If your local machine has Ansible installed, you can run the playbook locally using a ProxyJump through the bastion:

```bash
cd ansible

# Override SSH args to jump through bastion
ANSIBLE_SSH_ARGS="-o StrictHostKeyChecking=no -o ProxyJump=ec2-user@<BASTION_IP>" \
ansible-playbook -i inventory/hosts.ini playbook.yml
```

## SSH Reference

### Bastion ProxyJump to private instances

```bash
ssh -i your-key.pem \
    -J ec2-user@<BASTION_PUBLIC_IP> \
    ubuntu@<UBUNTU_PRIVATE_IP>      # Ubuntu instances use 'ubuntu' user

ssh -i your-key.pem \
    -J ec2-user@<BASTION_PUBLIC_IP> \
    ec2-user@<AMAZON_PRIVATE_IP>    # Amazon Linux instances use 'ec2-user'
```

### `~/.ssh/config` shortcut

```
Host bastion
  HostName <BASTION_PUBLIC_IP>
  User ec2-user
  IdentityFile ~/path/to/your-key.pem
  ForwardAgent yes

Host ansible-ctrl
  HostName <ANSIBLE_CONTROLLER_PUBLIC_IP>
  User ec2-user
  IdentityFile ~/path/to/your-key.pem
  ForwardAgent yes

Host ubuntu-*
  User ubuntu
  IdentityFile ~/path/to/your-key.pem
  ProxyJump bastion

Host amazon-*
  User ec2-user
  IdentityFile ~/path/to/your-key.pem
  ProxyJump bastion
```
